# Security Policy

## Supported Versions

Security fixes are handled on the `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for a vulnerability.

Use GitHub's private vulnerability reporting or open a private security advisory for this repository. Include:

- affected commit or version,
- clear reproduction steps,
- expected and actual behavior,
- impact assessment,
- any relevant logs or screenshots with secrets redacted.

## Scope

Security-sensitive areas include authentication, Turnstile/session handling, URL fetching, browser sandboxing, file uploads, archive extraction, static analysis, external command execution, storage access, and generated evidence artifacts.

Do not submit real credentials, private investigation material, malware samples, customer data, or unredacted secrets.
